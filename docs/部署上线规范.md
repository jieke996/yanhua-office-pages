# 部署上线规范

本项目为静态前端工具系统，线上发布前遵循通用部署上线规范，并结合本项目补充以下规则。

## 上线前确认

- 明确部署范围：默认只同步 HTML、CSS、JS、静态数据文件和必要资源文件。
- 确认是否影响接口配置：重点检查 `config.js` 是否变更。
- 确认是否影响线上业务数据：不得覆盖数据库、上传文件、日志、缓存或运行时生成文件。
- 准备回滚方式：保留上一个可用版本的静态文件，必要时回退到上一次提交或备份目录。
- 验证清单：登录页、首页工具导航、目标工具入口、目标工具页面、核心筛选交互、移动端布局。

## 默认排除

- 数据库文件
- 上传文件
- 日志文件
- 缓存文件
- 运行时生成的数据文件
- 本地临时文件，例如 `.DS_Store`

## 上线后验证

- 验证线上首页可访问。
- 验证目标工具入口位置正确。
- 验证目标工具页面非空白，筛选和重置可用。
- 验证浏览器控制台没有影响功能的错误。
- 将验证结果记录到 `docs/迭代更新记录.md`。
