# 学历提升学员系统 PRD 开发文档

## 1. 项目背景

言华办公系统目前已有“通辽国开专业查询”和“成考院校专业查询”等招生辅助查询工具，但 2025、2026 年国家开放大学和成人高考学员信息仍以 Excel 表维护。随着学员批次、费用、学校专业、推送机构和后续跟进信息增多，单纯表格不利于权限控制、数据查询、历史留存和导入导出。

本次拟在言华办公系统内新增“学历提升学员系统”，入口放在左侧导航“待办系统”下方，仅最高管理员可见和使用。

## 2. 产品定位

学历提升学员系统是面向最高管理员的内部学员档案管理后台，用于统一管理国家开放大学和成人高考学员，支持真实后端数据库的新增、编辑、删除、导入、导出、回收站和操作留痕。

系统第一版不做公开招生页面，不替代现有“专业查询”工具，而是承接已经成交或正在办理的学员管理工作。

## 3. 目标用户

- 最高管理员：可看到入口、访问页面、调用接口，并拥有学员新增、编辑、软删除、恢复、导入、导出、查看回收站等全部权限。
- 其他账号：左侧导航不展示“学历提升学员系统”，即使直接访问页面或接口也应被后端拒绝。

## 4. MVP 功能范围

第一版必须包含：

- 入口：左侧导航“待办系统”下方新增“学历提升学员系统”。
- 权限：仅最高管理员可见入口、可访问页面、可调用后端接口。
- 学员列表：支持分页、搜索、筛选、排序、快速查看详情。
- 新增学员：支持手动新增国开/成考学员。
- 编辑学员：支持编辑学员基础信息、报考信息、费用信息、备注、推送机构等字段。
- 删除学员：删除进入回收站，不做物理删除。
- 回收站：支持查看已删除学员、恢复学员；物理清理如需做，应单独二次确认。
- Excel 导入：支持导入 2025/2026 国开、成考历史表，提供预览、字段映射、校验、重复处理。
- Excel 导出：支持按当前筛选结果导出、全量导出、回收站导出。
- 操作日志：记录新增、编辑、删除、恢复、导入、导出等关键操作。

第一版暂不包含：

- 学员移动端小程序。
- 自动短信、自动缴费、自动教务系统对接。
- 非最高管理员的细粒度分权协作。
- 线上数据库迁移执行和数据导入执行，需在方案确认、备份确认后单独操作。

## 5. 业务范围

### 5.1 学历提升类型

- 国家开放大学，简称“国开”。
- 成人高考，简称“成考”。

### 5.2 层次

- 高起专。
- 专升本。
- 高起本。

系统可根据“原始学历 + 目标学历”自动推断层次：

| 原始学历 | 目标学历 | 推断层次 |
| --- | --- | --- |
| 高中 / 中专 / 初中 / 同等学力 | 专科 | 高起专 |
| 专科 | 本科 | 专升本 |
| 高中 / 中专 / 初中 / 同等学力 | 本科 | 高起本 |

当原始学历或目标学历不完整时，允许人工选择层次，并在导入预览中标记“需确认”。

### 5.3 年份与批次

- 年份：当前已提供 2025、2026 年数据，后续按年度扩展。
- 国开入学批次：每年 3 月、9 月两次入学。
- 成考批次：以年度成人高考招生为主，可记录成交日期和报考年度。

## 7. 核心数据字段

### 7.1 学员基础字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 姓名 | 是 | 学员姓名 |
| 联系方式 | 是 | 手机号或联系方式，导入时校验格式但不强制丢弃异常数据 |
| 业务类型 | 是 | 国开 / 成考 |
| 年份 | 是 | 2025、2026 等 |
| 入学批次 | 国开必填 | 3月 / 9月；历史数据不能确认时标记“待确认” |
| 原始学历 | 否 | 专科、高中、中专、初中等 |
| 目标学历 | 是 | 专科 / 本科 |
| 学历层次 | 是 | 高起专 / 专升本 / 高起本，可自动推断后人工修正 |
| 原学校 | 否 | 原毕业学校 |
| 原专业 | 否 | 原专业 |
| 意向专业 | 是 | 学员报考或意向专业 |
| 报考学校 | 是 | 学员报考学校 |
| 成交日期 | 否 | Excel 日期需统一转为 yyyy-mm-dd |
| 报名费 | 否 | 金额字段 |
| 尾款 | 成考适用 | 成考表存在，国开默认空 |
| 推送机构 | 国开新记录必填 | 吉林牛老师 / 盛世领航 / 弘达教育 |
| 备注 | 否 | 保留原备注 |
| 状态 | 是 | 在读、待入学、已推送、待确认、已删除等 |

### 7.2 推送机构

国开学员必须保留“推送机构”字段。第一版选项固定为：

- 吉林牛老师
- 盛世领航
- 弘达教育

历史导入规则：

- 备注包含“牛老师”时，推断为“吉林牛老师”。
- 备注包含“盛世领航”时，推断为“盛世领航”。
- 备注包含“弘达”时，推断为“弘达教育”。
- 无法识别时，标记为“待确认”，导入预览中提示人工补齐。

## 8. 页面结构

### 8.1 左侧入口

位置：言华办公系统左侧导航“待办系统”下方。

名称：学历提升学员系统。

可见规则：仅最高管理员可见。

### 8.2 系统首页

顶部展示核心统计：

- 学员总数。
- 国开学员数。
- 成考学员数。
- 本年度新增。
- 待确认字段数。
- 回收站数量。

### 8.3 学员列表页

列表字段建议：

- 姓名。
- 联系方式。
- 业务类型。
- 年份。
- 批次。
- 层次。
- 意向专业。
- 报考学校。
- 推送机构。
- 报名费。
- 尾款。
- 成交日期。
- 状态。
- 操作。

筛选条件建议：

- 关键词：姓名、电话、学校、专业、备注。
- 业务类型：全部、国开、成考。
- 年份：2025、2026、后续年份。
- 批次：3月、9月、待确认。
- 层次：高起专、专升本、高起本。
- 目标学历：专科、本科。
- 推送机构：吉林牛老师、盛世领航、弘达教育、待确认。
- 费用状态：报名费已填、尾款已填、费用未完善。
- 删除状态：正常、回收站。

### 8.4 新增/编辑抽屉

采用右侧抽屉或大弹窗，分组展示：

- 基础信息：姓名、联系方式、状态。
- 学历信息：原始学历、目标学历、层次。
- 报考信息：业务类型、年份、批次、意向专业、报考学校。
- 来源信息：原学校、原专业。
- 财务信息：报名费、尾款。
- 国开推送：推送机构。
- 备注信息：备注。

交互规则：

- 选择“国开”时显示并要求“推送机构”和“入学批次”。
- 选择“成考”时显示“尾款”。
- 修改原始学历或目标学历时自动重新推断层次，但允许人工覆盖。
- 保存前提示必填项和格式问题。

### 8.5 导入页

导入流程：

1. 选择业务类型：国开 / 成考。
2. 选择年份：2025 / 2026 / 其他。
3. 上传 Excel。
4. 系统识别表头和字段。
5. 展示导入预览。
6. 标记异常：手机号异常、层次无法推断、推送机构缺失、日期异常、金额异常、疑似重复。
7. 选择重复处理策略：跳过、覆盖、合并备注。
8. 确认导入。

### 8.6 回收站

回收站字段：

- 学员基础信息。
- 删除时间。
- 删除人。
- 删除原因。
- 操作：恢复。

第一版不建议提供直接物理删除。如果必须物理删除，应另设二次确认，并在部署文档和数据备份记录中明确。

## 9. UI 方案

### 9.1 参考截图拆解

- 当前办公系统是左侧导航 + 顶部用户栏 + 中央工作台布局。
- 导航项使用图标加文字，入口应跟“待办系统”同级，不应做成首页卡片入口。
- 首页卡片风格偏浅蓝、白底、轻边框、圆角和轻阴影。
- 学员系统属于后台业务工具，应以表格、筛选栏、抽屉、状态标签为主，避免营销式大首屏。

### 9.2 视觉风格

- 背景：延续现有浅蓝灰背景。
- 主色：延续系统蓝色，用于主按钮、选中态和关键数字。
- 辅助色：国开用蓝绿色标签，成考用橙色或紫色标签，回收站用灰红色警示。
- 表格：紧凑、可扫描，操作列固定在右侧。
- 弹窗/抽屉：字段分组清楚，避免一次性堆满。

### 9.3 状态设计

必须覆盖：

- 加载中。
- 空列表。
- 无权限。
- 导入预览。
- 导入成功。
- 导入失败。
- 表单校验错误。
- 回收站空状态。
- 接口错误。

## 10. 权限与安全

建议最高管理员识别采用其中一种方式：

方案 A：新增角色 `super_admin`。

- 优点：语义清晰，后端和前端判断简单。
- 缺点：需要调整用户表或认证返回结构。

方案 B：保留角色，新增权限位 `educationStudentManage: true`。

- 优点：与现有 `permissions.invitationAllViewMasked` 风格一致，改动较小。
- 缺点：如果未来最高管理员权限很多，权限位会增多。

推荐：第一版采用“权限位 + 后端强校验”，即 `permissions.educationStudentManage === true` 才能看到和访问；如果后续要建设完整后台角色体系，再升级为 `super_admin`。

必须做到：

- 前端隐藏入口只是体验优化。
- 后端接口必须校验权限。
- 导入、导出、删除、恢复必须写操作日志。
- 不得在前端代码中写死敏感账号、密码、Token。

## 11. 非功能需求

- 数据安全：线上数据库操作前必须备份。
- 性能：第一版 48 条历史数据很小，但表结构应支持未来数千条学员记录。
- 可维护性：字段应统一使用后端数据库字段，不依赖 Excel 列顺序。
- 可追溯：导入批次、源文件名、创建人、更新人、删除人都要保留。
- 可恢复：删除进入回收站，默认可恢复。
- 兼容性：桌面端优先，移动端保证可查看和基础操作。

## 12. 验收标准

开发完成后需验收：

- 最高管理员登录后，左侧“待办系统”下方显示“学历提升学员系统”。
- 普通用户登录后，看不到入口，直接访问页面显示无权限，调用接口返回无权限。
- 可新增一条国开学员，必填推送机构和入学批次。
- 可新增一条成考学员，可填写报名费和尾款。
- 可编辑学员信息并刷新后仍保留。
- 删除学员后，正常列表消失，回收站可见。
- 回收站恢复后，正常列表重新出现。
- 可导入 2025/2026 国开、成考 Excel，导入预览能识别字段差异。
- 导入后数据总量与确认导入条数一致。
- 可按业务类型、年份、层次、推送机构筛选。
- 可导出当前筛选结果为 Excel。
- 操作日志可记录关键行为。
- 页面完成桌面端和移动端截图自检，无明显遮挡、溢出、重叠和空白页。
