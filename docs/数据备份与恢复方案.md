# 数据备份与恢复方案

本项目当前以静态前端文件为主，功能数据主要来自后端接口或前端静态数据文件。

## 备份对象

- 线上静态页面文件。
- `config.js` 接口配置。
- `tools/` 下的工具页面和静态数据文件。
- 后端数据库、上传文件、日志、缓存等线上运行数据，如部署范围涉及这些内容，必须单独备份。

## 备份方式

- 代码文件优先通过 Git 提交记录保留版本。
- 上线前可复制当前线上静态目录为带日期的备份目录。
- 涉及数据库时，先导出数据库备份，再执行迁移或覆盖操作。

## 恢复方式

- 静态页面异常：回退到上一次可用提交或线上备份目录。
- 配置异常：恢复上一版 `config.js`。
- 数据异常：停止继续写入，使用上线前数据库或业务数据备份恢复，并复核核心数据量。

## 注意事项

- 严禁用本地测试数据覆盖线上业务数据。
- 严禁把账号、密码、API Key、Token、Cookie、私钥写入代码、日志或文档。
- 默认部署不包含数据库、上传目录、日志、缓存和运行时文件。
